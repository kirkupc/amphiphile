# Claude Code configuration

Project-level instructions for Claude Code sessions on this repository.

## Project context

logD7.4 prediction service: SMILES in, predicted logD + uncertainty + reliability flag out. LightGBM baseline + Chemprop D-MPNN ensemble. Scaffold-split evaluation on ChEMBL + ExpansionRx external benchmark.

## Development conventions

- Package manager: `uv`. All commands via `uv run`.
- Lint/format: `ruff check` + `ruff format`. Strict mypy on `src/`.
- Tests: `pytest -m 'not slow'` for fast suite (~3s), `pytest` for full (includes GPU/training tests).
- `make check` must pass before any commit: ruff + mypy + fast tests.
- Git: atomic commits with descriptive messages. Never force-push main.
- GitHub account: `kirkupc`.

## Code style

- Type hints everywhere. RDKit modules exempt from `disallow_untyped_calls` (no stubs).
- Minimal comments — only when the "why" is non-obvious.
- No unnecessary abstractions. Three similar lines > premature helper.
- Invalid SMILES return `None` at the feature level; callers mask.
- `FeatureSpec` dataclass controls feature blocks; serialised with model for train/inference consistency.

## Key architecture decisions

- Ensemble diversity via `feature_fraction=0.7, bagging_fraction=0.7` (not just seed variation).
- Conformal calibration uses absolute residuals, not Mondrian (Mondrian inflates on OOD).
- Reliability threshold calibration uses absolute Tanimoto grid, not val-set quantiles.
- Chemprop training requires GPU (Colab notebook). Local inference works on macOS ARM with safe collation.
- Henderson-Hasselbalch pKa corrections use group-average literature pKa values.

## What NOT to do

- Don't add dependencies without checking `pyproject.toml` first.
- Don't modify `models/` or `data/` contents directly — these are generated by training/data pipelines.
- Don't use `pytorch_lightning` — always `lightning.pytorch` (namespace clash causes silent failures).
- Don't use val-set quantiles for Tanimoto thresholds (produces thresholds that reject 99.5% of OOD data).
